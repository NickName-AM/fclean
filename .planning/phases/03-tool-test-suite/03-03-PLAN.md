## Trust Boundaries

This plan modifies `create_feature()` (adds a `dry_run` parameter, default False) and adds
tests. The dry_run path is strictly write-suppressing — it can only print, never write — so it
reduces rather than expands the filesystem-write surface. Test writes are confined to
`tmp_path` via `monkeypatch.chdir(tmp_path)`. No network calls, no new user-facing input
(the argparse surface is unchanged; the parameter is reachable only programmatically until
Phase 6 wires `--dry-run`).
