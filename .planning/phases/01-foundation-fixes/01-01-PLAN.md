Purpose: Generated Dart code currently contains syntactically valid but semantically wrong class names for any multi-word (snake_case) feature or entity name. Nothing downstream can be trusted until class names are correct.
Output: `fclean.py` with a module-level `to_pascal_case()` and zero remaining `.capitalize()` calls; `tests/test_utils.py` and `tests/test_generator.py` Wave 0 scaffolds proving the fix.
